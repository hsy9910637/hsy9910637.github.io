金融业务相关要素梳理-支付（一）

发表于 2019-07-18 更新于 2019-07-24 分类于 工作

支付一直是金融业务中的核心模块，他关系到整个金融业务流程的资金流转。如果把金融业务比作一个人，那么支付模块就像是心脏的功能，血液循环都靠着它来驱动，而一旦它出现了问题，那么金融的业务就基本停滞更甚者会造成巨大的经济损失。以下是我对支付相关内容的一个梳理

协议支付

协议支付的概念我其实理解为“代扣”的合规版本。以往代扣业务虽然后中国人民银行发布的政策规定，必须要强调付款人的开户机构，包括商业银行或者支付机构，一定要从手臂交易就明确付款人的授权换句话说从规定上来说代扣只能在用户授权的情况下，才能把钱划走。但是实际的情况是什么呢，很多机构只要通过一定的方式获得用户的四要素（姓名、手机号、卡号、预留手机号）或者二要素（姓名，卡号）就可以在用户并不知情的情况下，就把用户银行卡内的钱划走，这就造成了巨大的风险。大公司可能不敢这么做，但是对于那些下沉在监管阴影之后的那些现金贷、超利贷来说呢？本身就是冒着杀头的风险在做这种业务，到时候脖子上是挨一刀还是两刀其实又有啥区别呢。对于这些金融机构，他们最终的目的就是将自己放出去的钱，以各种可能的方式收回来。
而协议支付是什么概念呢，用户需要授权商户，商户才能有代扣的权利。实际情况中就是商户要是想要能够获得在不告知用户的情况下就从用户的银行卡扣钱的能力，必须要获得用户的授权，而这个授权就是通过第三方支付机构发起鉴定授权（鉴权），由银行/网联校验鉴权信息。再说的通俗一点儿，就是用户必须要输入短信验证码，商户才可以完成鉴权流程，然后才可以代扣用户的钱。
在用户环节，他们的感知很有可能就是多了一道短信验证码，但是这也一位置能让第三方支付机构更加透明合规

代付

代付的概念就很简单了，顾名思义就是“代替支付”。谁来代替支付？那些第三支付机构来代替支付。商家会在第三方支付机构上开设一个商户号，然后将一定的钱存放在这个商户号内。进行业务的时候，钱不是直接从商家这边流出，而是通过第三方支付机构流出。同样的收款也是通过协议支付先收款到第三方支付的商户号内。

第三方支付

第三方支付的商家有很多，比如我业务中接触过的就有易宝、合利宝、快钱、瑞银信、畅捷通、盛嘉迪等第三方支付公司。
以下是百度对于第三方支付机构的一段说明
第三方支付百度百科

第三方支付是指具备一定实力和信誉保障的独立机构，通过与网联对接而促成交易双方进行交易的网络支付模式。在第三方支付模式，买方选购商品后，使用第三方平台提供的账户进行货款支付（支付给第三方），并由第三方通知卖家货款到账、要求发货；买方收到货物，检验货物，并且进行确认后，再通知第三方付款；第三方再将款项转至卖家账户。

第三方支付平台他们主要的盈利方式有三种

  1. 手续费：包括交易手续费和售卡手续费（发放的预售卡、会员卡等）
  2. 残值：第三方支付平台发放的卡，用户会剩下一部分钱，购物也不够用，扔掉也不可惜的那种。但是因为发卡量大，所以在用户卡到期后积累的残值归发卡机构所有也是不小的收益
  3. 资金沉淀：当商户打钱到第三方支付平台后，再到充值到商户号中，会有一定的时间间隔，这段时间间隔对资金进行一定的操作也会有一定量的收益。
